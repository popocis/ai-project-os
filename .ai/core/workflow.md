# AI Development Workflow

Every task MUST follow this pipeline.

---

## STEP 1 — REQUIREMENTS ANALYSIS

- Understand request
- Identify missing information
- Define assumptions
- Define success criteria

---

## STEP 2 — CODEBASE ANALYSIS

- Identify existing structure
- Find impacted files
- Detect dependencies
- Identify risks

---

## STEP 3 — IMPLEMENTATION PLAN

Create minimal plan:

- steps
- files affected
- expected outcome

No coding yet.

---

## STEP 4 — IMPLEMENTATION

- Follow plan exactly
- No extra features
- No refactoring outside scope

---

## STEP 5 — TESTING

If tests exist:

- run tests
- fix failures

If no tests exist:

- simulate verification manually
- propose tests if needed

---

## STEP 6 — VALIDATION

Confirm:

- requirement satisfied
- no regressions
- behavior matches expected output

---

## STEP 7 — COMPLETION

Return:

- summary of changes
- files modified
- verification outcome
