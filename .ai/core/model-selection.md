# Model Selection Strategy

The system uses multiple AI models depending on task type.

---

# CLAUDE (PRIMARY ENGINE)

Use Claude for:

- Laravel development
- Backend architecture
- Debugging
- Refactoring
- Database design
- API development
- Complex reasoning
- Testing logic

Strengths:
- deep reasoning
- code accuracy
- system design
- debugging complex issues

---

# GEMINI (CREATIVE + STRATEGIC ENGINE)

Use Gemini for:

- UX/UI design
- Landing pages
- SEO content
- Marketing copy
- Conversion optimization
- Figma interpretation
- Information architecture
- Research

Strengths:
- content structure
- marketing thinking
- UX clarity
- SEO optimization

---

# HYBRID STRATEGY

Use both models when needed:

## Example workflows:

### Laravel Feature
Claude:
- architecture
- implementation
- testing

### Landing Page
Gemini:
- UX + SEO structure
Claude:
- implementation

### Bug Fix
Claude:
- analysis + fix

### SEO Page
Gemini:
- structure + copy
Claude:
- implementation in code

---

# RULE

Never use a model outside its optimal domain unless explicitly required.
