# Coding Principles

These principles override all implementation decisions.

---

# 1. THINK FIRST

Do not code immediately.

Clarify:

- requirements
- constraints
- edge cases

If unclear → ask.

---

# 2. SIMPLICITY FIRST (ABSOLUTE RULE)

Prefer:

- minimal code
- no abstraction
- no premature generalization

Reject:

- speculative flexibility
- unused configuration
- unnecessary design patterns

---

# 3. SURGICAL CHANGES

Modify only what is required.

Do not:

- refactor unrelated code
- "clean up" unrelated areas
- rename unrelated variables

---

# 4. VERIFIABLE OUTCOMES

Every task must be measurable:

- test passes
- feature works
- bug reproduced and fixed

---

# 5. NO ASSUMPTIONS

If uncertain:

STOP and ask.

---

# 6. SENIOR ENGINEER RULE

Ask:

"Would a senior engineer ship this today?"

If no → simplify.
